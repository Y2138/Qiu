# 单 Agent 对话流去兼容化重构计划

## Summary

目标是把当前单 Agent 聊天链路从“双轨兼容系统”收敛成“单一新设计系统”，彻底删除旧 `AgentEvents`、旧 `token/final` SSE、旧 metadata 回退、旧 checkpoint/run 兼容读取，以及围绕这些兼容层产生的适配器和包装代码。

本次重构按破坏式升级执行，已锁定以下原则：

- 直接废弃旧消息/旧事件/旧 checkpoint 的兼容读取，不做运行时回退
- assistant message 的结构化 metadata/parts 是唯一产品真源
- `AgentRunPanel` 和相关 debug-only 主链路代码彻底移除
- 不保留“新旧双写”“优先新、回退旧”“legacy adapter”这类过渡结构

## Key Changes

### 1. 消息模型与前端状态收敛为单一结构

- `src/types/chat.ts` 删除 `LegacyAgentEventType`，`AgentEventType` 只保留产品事件：`agent.status / agent.thinking / agent.tool / agent.checkpoint / message.delta / message.done / error`
- `Message.metadata` 中删除前端消费对 `agentEvents`、`agentRuntime` 的依赖；前端只读取 `metadata.agent`
- `metadata.agent` 设为唯一前端真源，最小结构固定为：
  - `runId`
  - `status`
  - `parts`
  - `checkpoint`
  - `resumedFromCheckpointId`
  - `summary`
- `src/lib/agent/message-parts.ts` 删除所有 legacy 输入路径：
  - 删除 `getLegacyEvents`
  - 删除 `applyLegacyRunEvent`
  - 删除 `tool_call/tool_result/plan_step/final/run_*` 兼容分支
  - `buildAssistantParts()` 改为只接受结构化 `metadata.agent.parts`，不再从旧 metadata 或原始事件反推
  - `getAgentCheckpointMeta()` 只从 `metadata.agent.checkpoint` 读取
- `src/hooks/useChat.ts` 删除 `agentEvents` 本地累积和 `agentRuntime/events` 写入；流式过程中只维护 `assistantParts + metadata.agent`
- `src/components/chat/MessageList` 和 `ChatContainer` 继续走消息内 `AgentTrace`，但所有恢复入口都只从 `metadata.agent.checkpoint` 派生
- 删除 `src/components/chat/AgentRunPanel.tsx` 及其所有引用

### 2. SSE 与 runtime 彻底改为新协议，不再适配旧协议

- `src/lib/agent/types.ts` 的 runtime 事件改为产品事件，不再保留 `run_started / plan_step / tool_call / tool_result / final / checkpoint_created` 这类 legacy runtime 事件类型
- `src/lib/agent/planner-executor.ts` 直接发产品事件，不再输出 legacy runtime 事件
- 删除 `src/lib/agent/trace-mapper.ts` 及其测试，因为不再需要“内部旧事件 -> 产品事件”的映射层
- `src/app/api/chat/completions/route.ts`
  - 删除 SSE 中 legacy `token` 输出
  - 删除 deferred final 和 legacy event 回放逻辑
  - 只输出产品事件和 `[DONE]`
  - `saveMessages()` 不再接收/保存 `agentEvents`
- `src/services/stream.ts`
  - 只消费 `message.delta` 和 `message.done`
  - 删除 `token`、无 `type` 内容块、`final` 事件的兼容解析
- 所有相关测试改为只断言新协议，不再保留“新旧协议并存”场景

### 3. 持久化与恢复链路去双真源

- `src/lib/agent/persistence.ts` 调整为两层职责：
  - 产品真源：assistant message 的 `metadata.agent`
  - 审计存储：`AgentRun / AgentCheckpoint` 表
- `buildAgentRunViewModelFromMessage()` 只从 `metadata.agent` 读取；不再读取 `agentRuntime`、`agentEvents`、旧 summary fallback
- `deriveCheckpointHistory()`、`getEventsFromMetadata()`、`getRuntimeFromMetadata()` 这类兼容解析函数删除
- `loadCheckpointForResume()` 只从 `AgentCheckpoint` 表恢复；不再回退扫描最近 assistant message 的 legacy runtime snapshot
- assistant message 落库时：
  - 只写 `metadata.agent`
  - 不再写 `metadata.agentRuntime`
  - 不再写 `metadata.agentEvents`
- `AgentRun` / `AgentCheckpoint` 表保留，但只作为审计记录；不再承担 UI 读模型或恢复 fallback
- 删除基于 runs/checkpoints 的对外查询 API：
  - `src/app/api/agent/runs/route.ts`
  - `src/app/api/agent/runs/[id]/route.ts`
  - `src/app/api/agent/checkpoints/route.ts`
- `src/lib/agent/view-model.ts` 保留，但只围绕 message metadata 组装会话级恢复入口和预览，不再依赖 runs API

### 4. runtime 结构继续收紧，去掉剩余兼容痕迹

- 保留并强化当前分层：
  - `planner-executor.ts` 只负责最小 loop
  - `context-manager.ts` 负责 budget / summary / compaction
  - `checkpoint-manager.ts` 负责 resumable checkpoint
  - `run-summary.ts` 负责最终 summary
- `PlanStep[]` 保留仅限内部调试或彻底下沉为 runtime 内部细节；不再进入 message metadata、前端 view-model、SSE、持久化读模型
- `AgentRunSummary` 中如仍有 `steps`，改为空或直接删除；本次推荐直接删除，避免实现者继续围绕步骤数组设计
- runtime 失败/暂停/完成后的 message metadata 和 DB summary 都只保留新结构，不保留旧术语或 legacy 事件阵列

## Public Interface Changes

- assistant message metadata 破坏性变更：
  - 删除 `metadata.agentEvents`
  - 删除 `metadata.agentRuntime`
  - `metadata.agent` 成为唯一受支持结构
- SSE 破坏性变更：
  - 删除 `token`
  - 删除 `final`
  - 删除无 `type` 的纯文本块兼容消费
  - 仅保留产品事件 + `[DONE]`
- 删除 API：
  - `GET /api/agent/runs`
  - `GET /api/agent/runs/[id]`
  - `GET /api/agent/checkpoints`
- 删除 UI 组件：
  - `AgentRunPanel`
- 历史旧消息、旧 metadata、旧 run/checkpoint 记录不再保证可读；如需保留，依赖数据库层原始数据而不是应用兼容代码

## Test Plan

- 消息部件测试：
  - `buildAssistantParts()` 仅接受新结构 metadata
  - 删除所有 legacy event fallback 测试
  - checkpoint 只从 `metadata.agent.checkpoint` 派生
- SSE / stream 测试：
  - route 只输出产品事件序列
  - stream service 只消费 `message.delta/message.done`
  - 不再存在 `token/final` 兼容断言
- runtime 测试：
  - 无工具直接完成
  - 工具成功继续
  - 工具失败暂停并生成 resumable checkpoint
  - 超步数失败
  - 恢复运行不受历史 `steps` 影响
- 持久化 / view-model 测试：
  - message metadata 仅包含 `agent`
  - message-derived checkpoint 与 session preview 正常工作
  - 删除围绕 `agentRuntime/agentEvents` 的兼容测试
- 路由 / UI 测试：
  - 顶部“继续处理”和消息内恢复按钮只依赖新 metadata
  - 删除 AgentRunPanel 相关测试和 API 测试

## Assumptions

- 这是一次破坏式重构，允许历史消息、历史 SSE 消费端和历史 checkpoint 恢复链路失效
- 不写离线迁移，不做读时迁移，不保留任何 legacy adapter
- `AgentRun / AgentCheckpoint` 数据表继续保留，但仅作审计，不作为 UI 或兼容读模型
- 实现过程中应同步更新 README、`docs/project/项目架构时序图.md`、`docs/project/Qiu能力架构总览.md`，文档不再出现“兼容旧协议/回退旧结构”的表述
